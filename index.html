Build an Agent Flow that calls a REST API

⏱ Estimated time: 45–60 minutes 📊 Level: Intermediate 🔑 Prerequisites: Copilot Studio license or trial
In this exercise you will build a Copilot Studio Agent Flow that calls the public Chuck Norris Joke REST API and returns a random joke to the user. You will create a Copilot agent, build the agent flow with an HTTP action, parse the JSON response, and connect everything together.

Architecture overview

The solution uses four components working in sequence:

👤 UserAsks for a joke
→
🤖 Copilot AgentUnderstands intent
→
⚡ Agent FlowHTTP GET request
→
🌐 Chuck Norris APIReturns JSON
TriggerUser asks agent: "Tell me a Chuck Norris joke"
AgentAgent recognises intent and invokes the registered agent flow as a tool
HTTPFlow sends GET https://api.chucknorris.io/jokes/random
ParseFlow extracts the value field from the JSON response
RespondFlow returns the joke text back to the agent, which forwards it to the user
💡 About the API https://api.chucknorris.io/jokes/random is a free, public API that requires no authentication or API key — perfect for demos and training exercises.

Step 1: Create a new Copilot agent

  1. In a web browser, navigate to Copilot Studio at https://copilotstudio.microsoft.com/ and sign in with your Microsoft 365 account when prompted.

    Note If the page shows a welcome tour, dismiss it before continuing.

  2. Verify the correct Environment is selected. The environment name is displayed in the top-right area of the header bar (e.g., Dev or your organisation's environment). If the wrong environment is shown, click it and select the correct one from the drop-down.

    Note If the top navigation bar does not load fully, press F5 to refresh your browser.

  3. On the Copilot Studio home page, locate the text field labelled Start building by describing what your agent needs to do and type the following prompt exactly:

    Tell me a random Chuck Norris joke from the internet

  4. Press Enter or click the Send (➤) button. Copilot Studio will use generative AI to auto-generate a name, description, and instructions for your agent.

    💡 Tip If you prefer to configure everything manually, click Skip to configure instead. You can then enter the name, description and instructions yourself in the next step.

  5. Review what was generated. The agent name, description, and instructions panel should now be populated. Do not click Create yet — you will refine the settings in Step 2.

Step 2: Configure agent settings

Whether you used the AI-generated agent or skipped to configure, verify and set the following fields on the agent overview / configure page:

FieldValue to enter
NameChuck Norris Joke Bot
DescriptionA fun agent that tells random Chuck Norris jokes using a REST API
InstructionsYou are a fun assistant. When the user asks for a joke, call the Chuck Norris flow to get a random joke and share it.
LanguageEnglish (United States) — default

  1. Click the Edit (pencil) icon next to the Instructions field if it is not already editable.

  2. Clear any auto-generated text and paste the following instruction exactly:

    You are a fun assistant. When the user asks for a joke, call the Chuck Norris flow to get a random joke and share it.

  3. Click Save.

    💡 Tip The Instructions field is the most critical setting — it tells the agent when and how to call the flow. Be specific. Vague instructions lead to unreliable triggering.

  4. Scroll down to the Knowledge section. Toggle Enable your agent to search all public websites to Off / Disabled so the agent relies only on the flow for its response.

  5. Click Create (or Save if the agent was already created). Wait for the agent to finish provisioning — this usually takes a few seconds.

Step 3: Create a new agent flow

  1. In the left navigation of your agent, click Actions (you may also see this labelled Tools in newer versions of Copilot Studio).

  2. Click + Add an action. A panel slides in from the right.

  3. Select New agent flow. Copilot Studio opens the Power Automate flow designer in a new browser tab (or embedded panel, depending on version).

    Note Agent flows are built directly inside Copilot Studio and use the standard Power Automate designer. The key difference is the trigger type used.

  4. The flow designer opens with the Run a flow from Copilot trigger already added. This trigger is mandatory for agent flows — do not delete or replace it.

  5. Click Save draft in the top-right corner. After saving, click the Overview tab at the top of the designer. Click the Edit button — a side panel opens where you can set the flow name and optionally add a description. Enter the name:

    Get Chuck Norris Joke

    Add a description if you like, then close or confirm the panel.

    💡 Tip Descriptive names make it much easier to find and manage flows later, especially in environments with many flows.

Step 4: Add an HTTP action to call the API

  1. In the flow designer, click + New step (or the + button below the trigger). The action search panel appears.

  2. In the search field, type HTTP and press Enter.

  3. From the results, select the HTTP action (listed under the Built-in or HTTP connector — the icon looks like a globe with angle brackets).

    ⚠️ Premium connector The HTTP connector is a premium connector. Ensure your environment has the appropriate Power Automate licence (e.g. Power Automate Premium, or a Power Apps Premium licence).

  4. Configure the HTTP action with these exact settings:

    FieldValue
    MethodGET
    URIhttps://api.chucknorris.io/jokes/random
    HeadersLeave empty
    BodyLeave empty — GET requests have no body
    AuthenticationNone
    💡 Fun fact The Chuck Norris API (api.chucknorris.io) is completely free and requires no API key or authentication — ideal for demos!

  5. Verify that the action card shows GET and the URL. The action is now ready.

Step 5: Parse the JSON response

The API returns a JSON object. You need to parse it so Power Automate can reference individual fields by name. Here is an example of a typical response body:

{
  "icon_url": "https://assets.chucknorris.host/img/avatar/chuck-norris.png",
  "id":       "abc123xyz",
  "url":      "https://api.chucknorris.io/jokes/abc123xyz",
  "value":    "Chuck Norris can divide by zero."
}

The joke text is in the value field. You will extract only that field.

  1. Below the HTTP action, click + and search for Parse JSON. Select the Parse JSON action (Data Operations category).

  2. In the Content field, click inside the field and then select Dynamic content. From the list under HTTP, select Body. This passes the raw HTTP response body into the parser.

    Note If you do not see Body in the dynamic content list, scroll down or search for body.

  3. In the Schema field, click Generate from sample. A dialog box opens.

  4. Paste the following sample JSON response into the dialog:

    {
  "icon_url": "https://assets.chucknorris.host/img/avatar/chuck-norris.png",
  "id": "abc123xyz",
  "url": "https://api.chucknorris.io/jokes/abc123xyz",
  "value": "Chuck Norris can divide by zero."
}

  5. Click Done. Power Automate generates the JSON schema automatically. You should see a schema with properties including icon_url, id, url, and value.

    💡 Tip You only need the value field for this exercise. The schema is still correct even though it includes extra fields — Parse JSON will simply make all of them available as dynamic content.

Step 6: Return the joke to the agent

The final step in the flow is to send the joke text back to the Copilot agent so it can present it to the user.

  1. Below the Parse JSON action, click + and search for Respond to Copilot. Select the action titled Respond to Copilot or Respond to agent (the name varies slightly by environment version).

    Note This action is specifically designed for agent flows. It does not appear in regular Power Automate flows and is only available because the flow uses the Run a flow from Copilot trigger.

  2. In the action, click + Add an output.

  3. Set the output type to Text (this maps to a string parameter).

  4. Set the output Name to joke.

  5. Click inside the Value field, then open Dynamic content. Under Parse JSON, select value. This maps the parsed value field (the joke text) to the output parameter.

  6. Verify the complete flow looks like this:

    TriggerRun a flow from Copilot
    HTTPGET https://api.chucknorris.io/jokes/random
    Parse JSONContent: body('HTTP') · Schema: auto-generated
    RespondOutput: joke = value from Parse JSON

  7. Click Save in the top-right corner of the flow designer. Wait for the save to complete (the button shows a spinner briefly).

    💡 Tip After saving, you can click Test (top-right) and select Manually to run the flow in isolation and verify the HTTP action returns a valid joke before connecting it to the agent.

Step 7: Connect the flow to your agent

  1. Return to the Copilot Studio browser tab where your agent is open. If the tab was closed, navigate to https://copilotstudio.microsoft.com/ and open your Chuck Norris Joke Bot agent.

  2. In the left navigation, click Actions (or Tools).

  3. Click + Add an action. In the panel that opens, you should see your newly saved Get Chuck Norris Joke flow listed. If it is not visible, search for it by name.

    Note Newly saved flows can take up to 30 seconds to appear in the list. If it does not appear, wait briefly and refresh the panel.

  4. Select Get Chuck Norris Joke from the list and click Add (or Next).

  5. You will be prompted to add a Tool description. This is the text the agent's AI uses to decide when to invoke the flow. Enter:

    Call this action when the user asks for a Chuck Norris joke
    ⚠️ Important The tool description is critical. The Copilot agent reads this description at runtime to determine which action to call. A vague or missing description means the agent may never trigger the flow. Be specific and use natural language that matches what users are likely to say.

  6. Verify that the flow output parameter joke is visible in the output mapping section. This confirms the connection between the flow's Respond to agent action and the agent.

  7. Click Save (or Finish) to confirm the action registration. The flow now appears in the Actions list of your agent.

Step 8: Test the agent end-to-end

  1. In the top-right corner of Copilot Studio, click Test your agent. The Test pane opens on the right side of the screen.

  2. In the chat input at the bottom of the test pane, type:

    Tell me a Chuck Norris joke

  3. Press Enter and observe the following sequence in the test pane:

    1Agent receives the message and identifies intent
    2Agent calls the Get Chuck Norris Joke flow (you may see a Running action… indicator)
    3Flow calls the API, parses the response, and returns the joke output
    4Agent presents the joke text to the user

  4. Verify that the agent's reply contains a joke. The response should look similar to:

    Here's one for you! 😎

    "Chuck Norris doesn't read books. He stares them down until he gets the information he wants."

  5. Send the same message again. You should receive a different joke each time, because the API returns a random joke on every call.

  6. Try an alternative phrasing to confirm the agent is robust to variation:

    Can you tell me something funny about Chuck Norris?
    💡 Tip The agent uses semantic matching — it does not need an exact phrase match. Any message that is close in meaning to "tell me a Chuck Norris joke" should trigger the flow.

  7. If the test passes, click Publish in the top-right to deploy your agent. After publishing, you can share it via Channels (e.g., Teams, a demo website, or a custom website embed).

    Note Publishing requires appropriate Copilot Studio licensing. If you are using a trial, publishing to the Demo Website channel is typically available.

Key takeaways

  • Agent flows use the Run a flow from Copilot trigger — this is mandatory and must not be removed.
  • The tool description on a registered action is what the agent's AI reads to decide when to invoke the flow. Write it clearly in natural language.
  • The HTTP connector is a premium connector — verify licensing before building in production.
  • Agent flows are deterministic — the same execution path every time, making them reliable for external API calls.
  • This same pattern works for any REST API: weather, internal CRM, ticket systems, stock prices, and more.
